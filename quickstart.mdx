You can go from zero to a fully running Book Paradise storefront in under five minutes. There is nothing to install, no terminal commands to memorize, and no accounts to create. Grab the three project files, open them in a browser, and you have a working luxury bookstore ready to customize.

## Launch Book Paradise

<Steps>
  <Step title="Download the three project files">
    Clone or download the repository to get the three files that make up Book Paradise:

    - `index.html` — all page markup, CDN link and script tags, and section structure
    - `style.css` — every custom style rule, CSS variable, and animation
    - `script.js` — the book data array, cart logic, wishlist logic, and all event listeners

    Place all three files in the same folder on your machine. The files must stay together — `index.html` references the other two by relative path.
  </Step>
  <Step title="Open index.html in a modern browser">
    Double-click `index.html` or drag it into the address bar of Chrome, Firefox, Safari, or Edge. The storefront loads immediately. You will see the hero section animate in, and the full product catalog renders from the JavaScript data layer — no network request needed for the book data itself.
  </Step>
  <Step title="(Optional) Serve with a local static server">
    Opening the file directly via `file://` works for almost everything, but a local HTTP server gives you more accurate behavior for features like the `fetch` API or service workers if you add them later. Spin one up with a single command:

    ```bash
    # Using Python (built into macOS and most Linux distros)
    python3 -m http.server 8080

    # Using Node.js / npx (no global install needed)
    npx serve .
    ```

    Then visit `http://localhost:8080` in your browser.
  </Step>
  <Step title="Explore the storefront">
    Browse the 24-book catalog in the **Products** section. Try the following to see all interactive features in action:

    - Type a title or author name into the search bar
    - Click a category chip to filter by genre
    - Use the sort dropdown to order by price, rating, or title
    - Add books to your cart and open the sliding cart sidebar
    - Heart-icon books to save them to your wishlist
    - Navigate to **Contact** and submit the form to see the SweetAlert2 confirmation
  </Step>
  <Step title="Customize for your own store">
    Making Book Paradise your own requires editing just two files:

    - **`script.js`** — find the `books` array near the top of the file. Each object holds `id`, `title`, `author`, `price`, `rating`, `category`, `cover`, and `description` properties. Add, remove, or edit entries to match your real inventory.
    - **`style.css`** — locate the `:root` block at the top of the file. Change the CSS custom property values (e.g. `--color-primary`, `--color-gold`, `--font-heading`) to retheme the entire site in seconds.
  </Step>
</Steps>

## What Loads from CDN

Book Paradise has no `node_modules` folder because every third-party library is loaded via a `<link>` or `<script>` tag in `index.html`. The following libraries are fetched from CDN on first page load and then cached by the browser:

| Library | Purpose |
|---|---|
| **Bootstrap 5** | Responsive grid, modal, offcanvas, and utility classes |
| **Bootstrap Icons** | Icon font used for cart, wishlist, and navigation icons |
| **Font Awesome** | Additional icon set used in the about and features sections |
| **AOS (Animate On Scroll)** | Scroll-triggered entrance animations on cards and sections |
| **Animate.css** | CSS animation utility classes for hero and modal transitions |
| **SweetAlert2** | Stylish popup dialogs for cart confirmation and form feedback |
| **Google Fonts** | Playfair Display (headings) and Poppins (body text) |

<Tip>
  You do not need to run `npm install` or any package manager command. All library URLs are already written into the `<head>` and end of `<body>` in `index.html`. Simply open the file and the browser fetches everything automatically on first load.
</Tip>

<Warning>
  Cart and wishlist data are stored in your browser's `localStorage`. This means the data is tied to the specific browser and device being used — it does not sync across devices. If a user clears their browser data or uses a private/incognito window, their saved cart and wishlist will not be present.
</Warning>
